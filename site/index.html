plumbline.

Reproduce paper numbers from 3D-vision foundation models.

Plumbline is a Python evaluation harness for monocular depth, multi-view depth, and pose-estimation models — VGGT, Depth Anything 3, MASt3R, Metric3D-v2, MoGe, Marigold, and others.

Each reproduction is a pinned, audited YAML config — model variant, dataset split, metric, alignment, crops, depth clip — that hits the number reported in the original paper. It exists because the published number is rarely what you get when you clone the model repo and run its eval script.

why/

why plumbline

Comparing modern 3D-vision models is harder than it should be: each repo ships its own eval script, each paper uses its own quietly different protocol, and just running a model produces numbers that don’t match the ones published with it. Plumbline closes that loop.

  1. 01

    Reproducible by default.

    Each entry is a pinned, audited YAML config that targets a specific paper number — model variant, dataset split, metric, alignment, crops, clips — and lands within a documented tolerance.

  2. 02

    One protocol across models.

    The same loader, the same alignment, the same crops are used for every model on every dataset. Differences in observed numbers stop being noise from incompatible eval code.

  3. 03

    The right defaults baked in.

    The silent protocol details that decide whether a number lands — alignment space, depth clipping, the variant of the dataset that the paper actually used — are pinned per config. You don’t have to rediscover them.
