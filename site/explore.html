verified/ provenance matrix

Model × benchmark, every number traced to its paper.

Every number is a plumbline run, shaded by performance — the greenest cell is the best result in its column (that benchmark); dimmer is weaker. Provenance rides the glyph, not the colour: ✓ verified numbers land within their pinned tolerance of the source paper, ℹ off-paper ones don't (paper-private recipes, protocol deltas, model limits — the hover names which), and ◇ have no paper target. Only the 36 verified are safe to cite. Switch the metric with the tabs; hover any number for observed vs paper, the deviation, and the citation. Each row follows its own paper's eval protocol, so read the shading as a within-column ranking, not a controlled head-to-head.

colour = performance weaker → best (within column) provenance — ✓ verified ℹ off-paper ◇ no paper target Δ vs paper on hover

Metrics are grouped depth · pose · trajectory. The pose and trajectory tabs include the companion numbers each paper row also reports (RRA@15 / RTA@15, RPE-trans / RPE-rot°), audited the same way; the “36 verified results” counts the primary metric per reproduction, not these companions. Tolerance is pinned per-cell: ±5% for 32 of the 36 cells, ±10% for the three video/trajectory-pose cells (sub-cm ATEs amplify small fp16-vs-bf16 drift), and ±2% for DA3 on NYU. Only one cell exceeds 5% — DAGE Sintel at +7.3%, inside its ±10% band. Cells outside their pinned tolerance don't count as verified and live in DISCREPANCIES.md. Every value is audited against the source PDF (table + column + row) — AUDIT.md. Some baselines are cited from a later paper's comparison table (e.g. DA-V2 / DUSt3R rows measured in MoGe / MASt3R) — the hover names the exact source.
