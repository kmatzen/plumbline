# Security Policy

## Supported versions

plumbline is pre-1.0; only the latest released version on PyPI
(`plumbline-bench`) receives fixes. Please upgrade before reporting.

## Reporting a vulnerability

Please report security issues privately rather than opening a public issue.

- Preferred: open a [GitHub private security advisory](https://github.com/kmatzen/plumbline/security/advisories/new).
- Alternatively, email **kmatzen@gmail.com** with `[plumbline security]` in
  the subject.

Include a description, reproduction steps, and the affected version. We aim to
acknowledge within a few business days. Because plumbline runs third-party
model code and downloads checkpoints/datasets, reports about unsafe
deserialization, arbitrary-code-execution paths in adapters, or supply-chain
concerns are especially welcome.
