### `da-v2-base-nyuv2` (7.0 % below paper ‚ÄĒ citation-level)

The YAML tags the citation as **UNVERIFIED**: the DA-V2 paper's Table 2

### `moge-vitl-nyuv2` (10.6 % below paper ‚ÄĒ protocol-level)

Observed **0.0305** vs cited **0.0341**. The MoGe paper Table 3 has two
NYU rows for ViT-L:

- affine-invariant *depth*: AbsRel 0.0297
- affine-invariant *disparity*: AbsRel 0.0341

Plumbline claims `scale_shift_robust` fits in inverse-depth space, so
the apples-to-apples paper column is the disparity row (0.0341). But
our observed **0.0305** is a hair *above* the depth-row number (0.0297).
Either (a) plumbline's ROE is stronger than MoGe's and beats their
disparity-row result, or (b) `scale_shift_robust` is accidentally
fitting in depth space. **Action:** check
`src/plumbline/scale_alignment.py::scale_shift_robust` to confirm it
inverts both pred and GT before the least-squares fit; if it does,
this is a real "plumbline ROE > MoGe ROE" result and deserves a YAML
note rather than a citation fix.

### `moge-vitl-diode-indoor` (5.9 % above paper ‚ÄĒ citation-level)

Observed **0.0424** vs cited **0.0400**. But the cited 0.0400 is the
MoGe paper's **combined** DIODE val number (indoor + outdoor averaged),
and this YAML runs **indoor-only**. Structurally mismatched: indoor is
the easier slice, outdoor is what drags the combined mean, so an
indoor-only observed against combined-val paper is not comparing the
same population. **Action:** either swap the paper citation to a per-
domain number if MoGe breaks it out (the paper's Table 3 may not), or
re-cite as an indoor-only extrapolation and widen tolerance with a
clear note ‚ÄĒ but the tolerance cap is now 5 % repo-wide, so the cleaner
fix is to demote this row from `source_confidence: verified_pdf`.

### `moge-vitl-diode-both` (188.6 % above paper ‚ÄĒ dataset-level)

Observed **0.1154** vs cited **0.0400**. Not a harness or citation bug;
the YAML's own note flags the root causes:

- **Depth clip mismatch.** plumbline caps aligned depth at
  `[1e-3, 50] m`; DIODE outdoor GT ranges to 300 + m. MoGe paper almost
  certainly clips to KITTI's 80 m or NYU's 30 m.
- **DIODE depth format.** MoGe's eval uses the `uint16 depth.png` +
  per-sample `segmentation.png` from their HF dataset
  `Ruicheng/monocular-geometry-evaluation`; plumbline's DIODE loader
  reads raw `.npy` depth.
- **Sky masking.** MoGe paper almost certainly masks sky (GT = invalid
  at 300 m / beyond sensor range); plumbline's current DIODE mask may
  retain true far returns that trip up scale + shift.

DIODE-indoor on the same predictions is 0.0424 (close to paper), so it
*is* the outdoor slice that's broken here. The indoor-only cell covers
the same ground more cleanly. **Action:** defer full outdoor fix to
v0.2 (already tagged in `docs/SESSION_HANDOFF.md ¬ß Deferred`), or demote
this YAML to an informational smoke test.
